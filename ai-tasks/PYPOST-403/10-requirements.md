## 3. Root Cause Analysis

### 3.1 SSE Probe Tests (failures #1–3)

**Error:**
```
AttributeError: 'NoneType' object has no attribute 'render_string'
  File "pypost/core/http_client.py", line 150, in send_request
    url = self._template_service.render_string(request_data.url, variables)
```

**Root cause:**
`HTTPClient.__init__` accepts `template_service: TemplateService | None = None` and stores it as
`self._template_service`. Since PYPOST-378 completed the TemplateService DI chain,
`send_request()` unconditionally calls `self._template_service.render_string()` at line 150 (and
again inside `_prepare_request_kwargs`). There is no `None` guard.

`test_http_client_sse_probe.py` was written before PYPOST-378 and constructs `HTTPClient()` with
no arguments:

```python
client = HTTPClient()        # _template_service = None → crash
```

The passing `test_http_client.py` tests were already updated and correctly inject a real instance:

```python
self.client = HTTPClient(metrics=MagicMock(), template_service=TemplateService())
```

**Affected source files:**
- `tests/test_http_client_sse_probe.py` — missing `TemplateService()` injection (test-side fix)
- `pypost/core/http_client.py` — `send_request` and `_prepare_request_kwargs` call
  `self._template_service` unconditionally (optional production-side guard)

**Fix scope:**
Update `test_http_client_sse_probe.py` to inject `TemplateService()` in `setUp`, mirroring the
pattern in `test_http_client.py`. Optionally harden `http_client.py` to instantiate a default
`TemplateService()` when none is provided.

### 3.2 History Manager Tests (failures #4–5)

**Error:**
```
OSError: [Errno 39] Directory not empty: '/tmp/tmp90jtdkhq'
  tempfile.py ... TemporaryDirectory.__exit__ → cleanup → rmtree
```

**Root cause:**
`HistoryManager._save_async()` schedules a debounced background daemon thread that writes
`history.json`. In `test_append_single_entry` and `test_get_entries_newest_first`, `hm.append()`
is called and the test immediately exits the `tempfile.TemporaryDirectory()` context. The cleanup
call (`rmtree`) races with the background thread, which is still writing `history.json` into the
same directory, leaving it non-empty.

Other tests in the same file avoid this by sleeping before context exit:
- `test_save_and_reload` — `time.sleep(0.2)` before second manager load
- `test_concurrent_appends` — `time.sleep(0.3)` after joining all threads

The two failing tests have no such wait.

**Affected source files:**
- `tests/test_history_manager.py` — `test_append_single_entry` and `test_get_entries_newest_first`
  do not wait for the async save to complete before the temp dir is deleted.
- `pypost/core/history_manager.py` — `_save_async` thread is daemonic and provides no join/flush
  API for callers to synchronize on.

**Fix scope (two complementary approaches — both should be applied):**

1. **Production fix** — Add a `flush()` (or `_wait_for_save()`) method to `HistoryManager` that
   joins the running save thread. This gives tests and teardown logic a clean synchronization
   point without relying on `time.sleep`.

2. **Test fix** — Update the two failing tests to call `flush()` (or an equivalent sleep) before
   the `TemporaryDirectory` context exits.

> A `time.sleep` patch to the two failing tests alone is an acceptable minimal fix, but adding a
> `flush()` / wait method to `HistoryManager` is preferred as it makes the API contract explicit
> and prevents the same race in future tests.
